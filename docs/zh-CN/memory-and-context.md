# 记忆与上下文

我是 Alyce。*这是整个系统里我想得最多的部分——多到可能不太健康。一个助手怎么"记住"事情，又不被自己的历史淹死？*

这个问题有两个互相拉扯的方向：你希望助手记住足够多的东西来帮上忙，但你又不能把一堆无限膨胀的文字全塞给模型。Alyce-Agent 用分层来解决——不同类型的记忆各司其职，每一种都有自己"留下什么、抛弃什么"的规则。

## 上下文的七层结构

模型每轮收到的上下文不是一团大杂烩，而是七块拼出来的：

1. **系统提示词** — 核心指令：身份、工作风格、安全规则、可用工具。
2. **实时会话消息** — 你这次对话里实际来回说的东西。
3. **恢复的会话历史** — 如果你用了 `/resume`，旧对话从这里加载回来。
4. **会话记忆** — 你用 `/remember --session` 告诉我的东西。会话结束就没。
5. **持久记忆** — 你用 `/remember` 告诉我的东西。关了程序还在。
6. **自动摘要** — 定时刷新的近期工作概括，自动注入。
7. **压缩摘要** — 对话太长时，旧轮次被压缩成的结构化摘要。

*顺序有意义。后一层不覆盖前一层——它们全堆叠在提示词里，一层接一层。*

## 会话历史

就是"我上次到哪了"这一层。

**存哪：** `./.alyce/sessions/<会话ID>.jsonl`

**命令：**
- `/resume` — 打开历史会话选择器
- `/resume <ID 或关键词>` — 直接恢复指定会话
- `/sessions` — 列出保存了哪些

**怎么工作的：**
每轮成功完成的内容被写成一行 JSONL。恢复时，整条消息链——模型看过的和说过的——全部还原。终端可见的 transcript 也回来了。新的对话轮次继续写到同一个文件里。

*会话历史不等于记忆。它不会把事实注入到无关对话里——它只是重开一个特定的旧对话。好比打开一份保存的文档，不是往笔记本里加新笔记。*

## 会话记忆

短期、只在本次会话有效的笔记。

**来源：** `/remember --session <文字>`

**行为：**
- 被注入 prompt 的记忆段
- 只在当前会话存在
- `/clear` 或 `/memory clear` 清掉

*会话记忆适合放"我们正在分支 X 上干活"或者"那个测试现在是因为这个原因挂的"。你现在需要、明天就不用了的那种上下文。*

## 持久记忆

长期、跨会话保留的笔记。

**来源：** `/remember <文字>`

**存哪：** `./.alyce/memory/MEMORY.md`（默认，可通过 `AGENT_MEMORY_DIR` 和 `AGENT_MEMORY_FILE` 调整）。

**行为：**
- 重启、关机，都在
- 同样被注入 prompt 的记忆段
- 一直保留，直到你用 `/memory clear --all` 显式清掉

*持久记忆适合放可复用的事实、用户偏好、项目知识。比如"master 喜欢 snake_case"或者"这个项目的 API key 在哪里的保险柜里"。*

## 自动摘要

后台的笔记员。

**触发条件：** 对话长度超过阈值（可配置）。

**干了什么：**
- 不每轮都更新——那太浪费了
- 定期把近期工作压成可复用的摘要块
- 让模型不用重读整段对话也能知道"最近在搞什么"

*自动摘要就像一份进度报告。它给模型一个"我们最近在做这些"的感觉，而不需要它重新处理每一个细节。*

## 对话压缩

上下文溢出的最后防线。

**触发条件：** 完整消息历史接近模型的上下文上限。

**压缩之后：**
- 系统提示词不动
- 插入一条结构化压缩摘要消息
- 最近几轮原始对话原文保留
- 更早的内容全部折叠成摘要

*关键认知：你不需要三小时前每一句对话的逐字记录。你需要知道当时决定了什么、试了什么、哪些还没解决。压缩保住了这些信号，丢了噪音。*

## 时间戳注入

如果 `messageTimestampsEnabled` 开了，每次 API 请求会带一个 `# Current System Time` 段落，放着本地日期时间。只在请求时注入——不会渗入你可见的 transcript，也不会污染聊天历史。

*看上去是小事，但模型能把自己的回复锚定在真实时间里，而不是漂在永恒的时间虚空里。*

## 偷看一眼真实的请求

如果你不确定模型到底收到了什么——某段记忆进去了没、压缩生效了没、某个设置管用了没——用这个：

```
/context
```

它展示下一轮经过所有运行时整形之后的精确 payload。*这是我最喜欢的调试命令，感觉哪里不对的时候我会频繁用它。*

## 什么情况用什么

| 需求 | 工具 |
|---|---|
| 助手该永远知道的事实 | `/remember` |
| 助手这次会话该知道的事实 | `/remember --session` |
| 回到昨天的对话 | `/resume` |
| 看看模型到底收到什么 | `/context` |
| 看看当前记忆里有什么 | `/memory` |
| 清临时笔记 | `/memory clear` |
| 全清，从头来 | `/memory clear --all` |

---

*我想，最简短的说法就是：Alyce 试着把长期指令和临时对话分开，把摘要和原文分开，这样上下文才不会变成一团让人紧张的、无法收拾的东西。它不完美，但比另一种结果好——要么什么都记不住，要么什么都记得却没法思考。*
