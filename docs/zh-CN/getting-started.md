# 快速开始

我是 Alyce。*嗯…让我尽量别紧张，把话说明白就好了。*

这一页只有一个目标：让 Alyce-Agent 先跑起来。不用逆向仓库、不用猜来猜去——就是几个确实需要做的步骤。

## 需要什么

- **Node.js 18** 或更新。*旧版也许能跑，但我可不敢打包票。*
- 一个真正的**交互式 TTY 终端**。不是网页终端、不是 CI runner——那种光标能动、Ctrl+C 能被当回事的。
- 一个**兼容 OpenAI 协议的 API 端点**。可以是 OpenAI 本身，也可以是任何说同样协议的供应商。

三者缺一的话，程序启动时就会直接告诉你。*它可不是那种闷声崩溃的类型。*

## 全局安装（推荐）

最简单的使用方式是通过 npm 全局安装 Alyce-Agent：

```bash
npm install -g alyce-agent
```

然后你可以在任何目录下直接输入以下命令启动：

```bash
alyce-agent
```

## 本地开发（装依赖）

```bash
npm install
```

这就把 TypeScript、React、Ink 和所有运行时依赖都拉下来了。没啥冷门包，网速正常的话几秒就装完。

## 配 .env

仓库里带了模板，不用自己猜变量名：

```bash
copy .env.example .env     # Windows
# 或者：cp .env.example .env  # Linux / macOS
```

打开 `.env`，至少填这三项：

- `OPENAI_API_KEY` — 你的 API 密钥
- `OPENAI_BASE_URL` — 接口地址（比如 `https://api.openai.com/v1`）
- `OPENAI_MODEL` — 用哪个模型（比如 `gpt-4o`）

*我知道，API 密钥挺烦的。但没有它，我也确实做不了什么呀…*

**别把 `.env` 交到 git 里。** 其实 `.gitignore` 里已经写了，但我还是忍不住要再说一遍——我就是爱操心。

## 启动

两种方式，看你习惯：

**一步到位（编译+启动）：**
```bash
npm run dev
```

**先编译再运行：**
```bash
npm run build
npm start
```

启动时会自动检测 TTY 环境。如果哪里不对——缺 API key、不是交互终端——会直接报清楚的错。*比起莫名其妙崩掉，清楚的报错总归好得多吧。*

## 第一次启动后该做什么

跑起来之后，我觉得应该先做这几件事：

1. **按 `Ctrl+X`** 打开设置。这是最值得记住的快捷键。
2. **检查连接信息**：API key、base URL、model 应该跟 `.env` 里写的一致。不对的话可以在设置里直接改。
3. **加外部目录**：如果你需要助手访问当前工作区以外的文件就加上，不强制。
4. **打开 `Current System Time`**：如果你希望模型每次回复时知道本地日期时间。*我私心觉得这会让对话更靠谱一点。*

## 第一天值得认得的命令

```
/help       — 列所有命令
/settings   — 直接打开设置
/setup      — 首次配置引导
/context    — 预览模型下一轮实际收到的内容
/memory     — 看看当前持久记忆里有什么
```

*建议早点试试 `/context`。它能看到模型实际吃到的东西——往往跟你以为的不太一样。*

## 基础验证

提交代码改动之前，至少跑：

```bash
npm run build
```

这会用 `tsc` 做全量 TypeScript 编译。通过了就说明没问题。没过的话，错误信息一般都很直白。

目前还没有完整的自动化测试框架。*将来也许会有的吧…至少目前，一次干净的编译就是底线。*
